# ECS 项目修复记录 2026-07-10

## 本次修复

### 1. `human/systems/action/socialize_system.py`
- **问题**：`MemoryManagementSystem.record_person()` 参数顺序错误，把 `current_time` 放在了 `relationship` 的位置，导致 `multiple values for argument 'relationship'`。
- **修复**：改为关键字参数：`time=current_time`，`relationship="friend"`。

### 2. `civilization/systems/technology_evolution_system.py`
- **问题**：`_attempt_teach()` 中把 `recipe["materials"]` 当作 dict 调用 `.keys()`，且调用组件实例的 `record_attempt()`（组件已纯数据化，无此方法）。
- **修复**：
  - 解析 `recipe["materials"]` 字符串键（形如 `"wood:1.0+stone:2.0"`）为 `Dict[str, float]`。
  - 改为调用 `CraftingKnowledgeSystem.record_attempt()` 静态方法。

### 3. `civilization/systems/trade_system.py`（最小可用化）
- **问题**：使用不存在的 `ActionStatus.IN_PROGRESS` / `COMPLETED`、动态属性 `trade_partner` / `offered_resources` / `requested_resources`（slotted 组件无法动态挂载）、访问不存在的 `InventoryComponent` 资源字段、调用不存在的 `MemoryComponent.add_experience` 等。
- **修复**：
  - 使用 `RUNNING` -> `SUCCESS` 状态流。
  - 使用 `action.target_entity` 作为交易伙伴。
  - 简化为象征**交易：增加双方财富、加深关系、记录交易历史，避免真实物品转移带来的复杂度。

### 4. `decomposer/systems/decomposer_system.py`
- **问题**：对 `slots=True` 的 `DecompositionComponent` 使用 `__dict__` 访问/修改字段，导致 `AttributeError`。
- **修复**：使用 `getattr` / `setattr`。

### 5. `track_human_demo.py`
- 保留并确认命令行参数 `--human-count`、`--max-steps`、`--delta-hours` 已可用。
