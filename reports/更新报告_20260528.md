## 一、P0 严重问题修复

### ✅ P0-01: `System._update()` 死代码

| 项目 | 内容 |
|------|------|
| **文件** | `core/system.py` |
| **改动** | 删除未使用的 `_update()` 死代码方法；将 `update()` 文档注释更新为"子类应直接重写此方法"，签名统一为 `update(self, world: World, dt: float = 1.0)` |
| **原因** | 代码库中 50+ 个子类全部直接重写 `update()` 而非 `_update()`，模板方法模式从未生效。删除死代码使基类设计与实际使用一致。 |

### ✅ P0-02: 双调度器架构冲突

| 项目 | 内容 |
|------|------|
| **文件** | `main.py` (`SimulationLoop._init_systems` + `update`) |
| **改动** | 所有 System 在初始化时通过 `world.add_system()` 注册并设置 `priority`；`SimulationLoop.update()` 简化为统一调用 `self.world.update(delta_hours)` + `_regenerate_resources()` |
| **优先级设计** | 空间(0) → 时间(5) → 环境管线(20) → 人类(30) → 部落(39) → 生理(40) → 生物(50) → 规则(60) → 文明(70) |
| **影响** | 新增 System 只需 `world.add_system()` 即可被调度，无需修改 `SimulationLoop`，恢复了 ECS 可插拔性 |

**兼容性处理：**
- `SpaceSystem` 保持独立类（避免循环导入），但 `update()` 签名改为 `update(self, world=None, dt=None)` 以兼容 `world.update()` 调用
- `EnvironmentPipeline` 继承 `System`，`update()` 签名已兼容
- `CivilizationSystem` 内部子系统保持私有，不重复注册到 world

### ✅ P0-03: biology `try/except TypeError` 隐藏 API 不匹配

| 项目 | 内容 |
|------|------|
| **文件** | `main.py` 第 254-258 行 |
| **改动** | 删除 `try/except TypeError` 降级调用，统一为 `system.update(self.world, delta_hours)` |
| **验证** | 所有 12 个 biology system 的 `update()` 签名已统一为 `(self, world, dt)`，运行 300 步未触发 TypeError |

### ✅ P0-04: `except: pass` 吞噬异常

| 项目 | 内容 |
|------|------|
| **文件** | `human/systems/social/reproduction_system.py` 第 108-112 行 |
| **改动** | 裸 `except:` 改为 `except Exception:`，并将 `parent_identity` 显式置为 `None` |
| **影响** | 异常不再被静默吞噬，同时避免未定义变量引用错误 |

## 五、架构改进总结

| 改进项 | 修复前 | 修复后 |
|--------|--------|--------|
| System 注册 | 仅 SpaceSystem + TribeSystem 注册到 world | 全部 40+ System 注册到 world，按 priority 调度 |
| 调度方式 | `SimulationLoop` 手动遍历 8 组 | `world.update(delta_hours)` 统一调度 |
| 新增系统 | 需修改 `SimulationLoop.update()` | 仅需 `world.add_system()` |
| 日志输出 | 10+ 处 `print()` 散落于热路径 | 统一 `logging`，可分级控制 |
| 层间耦合 | `biology/DeathSystem` 直接导入 human 组件 | 拆分为 `biology/DeathSystem` + `human/HumanDeathSystem` |
| 死代码 | `_update()` 方法从未被调用 | 已删除，文档与实际一致 |
