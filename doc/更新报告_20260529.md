## 三、P2 中期架构升级（5 项）

### P2-1: PathfindingSystem（A* 寻路）

| 项目 | 内容 |
|:---|:---|
| **新建** | `human/systems/navigation/pathfinding_system.py` |
| **算法** | A* 寻路，四邻域移动，欧几里得距离启发函数 |
| **特性** | 动态障碍物检测（查询 SpaceSystem 实体位置）、目标被占时找最近可达点 |
| **注册** | `main.py` priority = 28（CombatAISystem 之前） |

### P2-2: MemoryDecaySystem（记忆衰减）

| 项目 | 内容 |
|:---|:---|
| **新建** | `human/systems/cognitive/memory_decay_system.py` |
| **机制** | 每 tick 扫描 `MemoryComponent`，按重要性衰减，低于阈值删除，超上限删除最旧 |
| **注册** | `main.py` priority = 43（HealthcareSystem 之后） |

### P2-3: 巨型 System 拆分

**WeatherEffectSystem → 5 个独立 System：**

| 新 System | 职责 | 常量提取 |
|:---|:---|:---|
| `HeatEffectSystem` | 高温 >30°C：口渴↑、疲劳↑、>35°C 热伤害 | `TEMP_THRESHOLD_MILD=30.0`, `HEAT_DAMAGE_RATE=0.1` |
| `ColdEffectSystem` | 低温 <10°C：精力↓、<0°C 冻伤 | `TEMP_THRESHOLD_SEVERE=0.0`, `COLD_DAMAGE_RATE=0.05` |
| `RainEffectSystem` | 降雨 >10mm：舒适度↓ | `RAIN_THRESHOLD=10.0`, `COMFORT_PENALTY=0.5` |
| `WindEffectSystem` | 风速 >10m/s：疲劳↑ | `WIND_THRESHOLD=10.0`, `FATIGUE_RATE=0.3` |
| `HumidityEffectSystem` | 湿度 <30%：口渴↑、>90%：舒适↓ | `HUMIDITY_LOW=0.30`, `HUMIDITY_HIGH=0.90` |

**ConflictManagementSystem → 2 个独立 System：**

| 新 System | 职责 | priority |
|:---|:---|:---:|
| `ConflictDetectionSystem` | 检测负面关系，创建冲突，强度高时触发 ATTACK | 31 |
| `ConflictResolutionSystem` | 管理冲突生命周期，策略推荐，低强度冲突自动衰减 | 32 |

### P2-4: ReputationComponent 业务逻辑迁移

| 项目 | 内容 |
|:---|:---|
| **新建** | `human/systems/social/reputation_system.py` |
| **迁移** | `adjust_reputation`/`get_status`/`spread_rumor`/`is_honorable`/`get_title` 等 → `ReputationSystem` 静态方法 |
| **组件** | `ReputationComponent`/`FameComponent`/`SocialStandingComponent` 纯数据化 `@dataclass(slots=True)` |
| **注册** | `main.py` priority = 35 |

### P2-5: 魔法值提取为常量

所有新建系统（5 个天气子系统 + 2 个冲突系统）均使用模块级常量定义，替换硬编码数值。

## 七、架构改进总结

| 改进项 | 修复前 | 修复后 |
|:---|:---|:---|
| 系统总数 | 50 | **63** |
| 天气影响 | `WeatherEffectSystem` 单系统处理 5 种效果 | 拆分为 **5 个独立 System**，单一职责 |
| 冲突管理 | `ConflictManagementSystem` 334 行，检测+解决混合 | 拆分为 **ConflictDetectionSystem + ConflictResolutionSystem** |
| 路径规划 | 无，实体直线移动 | `PathfindingSystem` A* 寻路，考虑动态障碍物 |
| 记忆管理 | 只增不减，内存膨胀 | `MemoryDecaySystem` 按重要性/时间衰减 |
| 经济系统 | `MoneySystem` 游离 ECS | `EconomySystem` + `WalletComponent` + `MarketPricesComponent` |
| 声誉系统 | Component 含业务逻辑 | `ReputationSystem` 接管，Component 纯数据化 |
| 组件目录 | 45 个组件全在 `human/` 下 | 通用组件迁移到 `core/`(6) + `biology/`(11) |
| 序列化安全 | `ReputationComponent.to_dict()` 返回 `{}` | `@dataclass(slots=True)` 正常序列化 |
| 异常处理 | `HumanDeathSystem` 静默吞异常 | 记录 WARN 日志 |
| 数值安全 | `health.hp` 可能超过 `max_hp` | `clamp(0, max_hp, hp)` |
