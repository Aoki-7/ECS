# ECS 项目长期记忆

## 项目概况

- **路径**：`D:\个人助手\workspace\ECS`
- **类型**：自研 ECS（Entity-Component-System）世界模拟系统，WorldBox-like 但更高精度
- **当前版本**：v4.15.1
- **规模**：~730 Python 文件 / ~78k 行 / 661 测试全过（646 常规 + 15 Phase 3组件测试 + 10 慢测试 + 41 子测试） / 180+ System
- **架构文档**：`ARCHITECTURE.md`（全分层架构梳理）

## 关键设计原则

- 无硬编码、从实践中学习
- 文明差异化、技术可失传
- 主客观分离、信息损失机制
- 组件纯度优先：业务逻辑迁回 System

## 重要架构决策

- `get_components()` 统一返回 list
- `World` 兼容 v3.9 API
- `FullSimulationLoop` 初始化使用 `init()` 重写
- 生理系统合并，删除重复拆分系统
- `MemoryComponent` 保留原调用签名，方法迁移到 System 静态方法
- `EnvironmentComponent` 从全局迁移到 per-cell 空间网格（100×100）
- 环境实现方向：全球基准 + 局部增量修正
- 重型网格系统缓存：`FireSpreadSystem`、`GasDiffusionSystem`
- `KnowledgeComponent` 必须挂载到人类实体
- 文明阶段可观测化：降低阶段阈值、提高技术水平增量
- 技术效果通过全局修正组件 `TechnologyModifierComponent` 共享
- 意图→行动补齐：`SOCIALIZE` / `TRADE` 必须由 `ActionPlanner` 生成行动队列
- 死亡前数据快照：观察者脚本每步预先保存记忆/关系/技能状态
